<task type="auto">
  <name>Task 2: Initialize Post @Version field and update seed SQL</name>
  <files>aini-inu-backend/src/main/java/scit/ainiinu/community/entity/Post.java, aini-inu-backend/src/main/resources/db/seed/10_core_sample_seed.sql, aini-inu-backend/src/main/resources/db/seed/20_status_edge_seed.sql</files>
  <action>
    **Root cause:** `Post.java` line 44 declares `@Version private Long version;` without initialization. Seed SQL INSERTs omit the `version` column, leaving it NULL in the database. When Hibernate performs an optimistic lock UPDATE (`WHERE version = ?`), null Long unboxing causes NPE (comment creation) or transaction commit failure (like toggle).

    **Fix 1 — Post.java:**
    Change line 44 from:
    ```java
    @Version
    private Long version;
    ```
    To:
    ```java
    @Version
    private Long version = 0L;
    ```
    This ensures newly created posts (via `Post.create()`) always start with version 0. Existing seeded rows still need SQL fix below.

    **Fix 2 — 10_core_sample_seed.sql:**
    In the `INSERT INTO post` block (around line 400), add `version` to the column list and `0` to each row value tuple. Also add `version = EXCLUDED.version` to the ON CONFLICT UPDATE clause.

    Change columns from:
    ```sql
    INSERT INTO post (
        id, author_id, content, like_count, comment_count, created_at, updated_at
    ) VALUES
    ```
    To:
    ```sql
    INSERT INTO post (
        id, author_id, content, like_count, comment_count, version, created_at, updated_at
    ) VALUES
    ```
    Add `0` before each `NOW()` timestamp pair in each row. For example:
    ```sql
    (3501, 1, '오늘 한강 산책 다녀왔어요. 날씨가 정말 좋네요!', 4, 3, 0, NOW() - INTERVAL '9 hour', NOW()),
    ```
    Add to ON CONFLICT clause:
    ```sql
    version = EXCLUDED.version,
    ```

    **Fix 3 — 20_status_edge_seed.sql:**
    Same pattern as Fix 2. In the `INSERT INTO post` block (around line 247), add `version` column and `0` values. Update ON CONFLICT clause.

    Change columns from:
    ```sql
    INSERT INTO post (
        id, author_id, content, like_count, comment_count, created_at, updated_at
    ) VALUES
    ```
    To:
    ```sql
    INSERT INTO post (
        id, author_id, content, like_count, comment_count, version, created_at, updated_at
    ) VALUES
    ```
    Add `0` before timestamps in each row:
    ```sql
    (9801, 7, '상호작용 없는 기본 게시글입니다.', 0, 0, 0, NOW() - INTERVAL '4 hour', NOW()),
    (9802, 6, '이미지 5장 경계 테스트 게시글입니다.', 5, 1, 0, NOW() - INTERVAL '3 hour', NOW())
    ```
    Add to ON CONFLICT clause:
    ```sql
    version = EXCLUDED.version,
    ```

    **IMPORTANT:** For existing database rows already containing NULL version, the user must run this SQL once:
    ```sql
    UPDATE post SET version = 0 WHERE version IS NULL;
    ```
    Or restart with fresh seed data. The entity fix only prevents future NULL versions; existing rows need the seed SQL or manual UPDATE.
  </action>

- Remove `import { FeedPostType, CommentType } from '@/types'` and `import { postService } from '@/services/api/postService'`.

- Update `FeedItemProps`: `{ post: PostResponse; currentUserId?: number; onDelete?: (postId: number) => void; onLikeUpdate?: (postId: number, liked: boolean, likeCount: number) => void; }`.

**Field name mapping (apply throughout JSX):**
- `post.images` -> `post.imageUrls`
- `post.images[0]` -> `post.imageUrls[0]`
- `post.likes` -> `post.likeCount`
- `post.comments` (count) -> `post.commentCount`
- `post.isLiked` -> `post.liked`
- `post.author?.avatar` -> `post.author?.profileImageUrl`
- `post.caption` -> `post.content`
- `post.location` -> remove (no location in PostResponse)
- `post.time` -> compute relative time from `post.createdAt` using a simple helper (e.g., minutes/hours/days ago in Korean)
- `post.author?.handle` -> remove (no handle in Author)
- `post.tags` -> remove tags section entirely (no tags in PostResponse)
- `post.author?.id` comparisons: both sides must be `number` type
- `cmt.author?.avatar` -> `cmt.author?.profileImageUrl`

**Internal state:**
- `const [post, setPost] = useState<PostResponse>(initialPost)` -- use PostResponse type.
- `const [isLiked, setIsLiked] = useState(initialPost.liked)` -- use `.liked` not `.isLiked`.
- `const [comments, setComments] = useState<CommentResponse[]>([])` -- start empty, load on expand.
- Remove `initialPost?.commentsList` fallback (PostResponse has no embedded comments).

**isMyPost check:**
- `const currentUserId = Number(useUserStore.getState().profile?.id)` -- or use the prop.
- `const isMyPost = !!(currentUserId && post.author?.id === currentUserId)`.

**Optimistic like (DEC-018 pattern):**
- Save previous state: `const prevLiked = isLiked; const prevCount = post.likeCount;`
- Optimistic update: `setIsLiked(!isLiked); setPost(prev => ({ ...prev, likeCount: isLiked ? prev.likeCount - 1 : prev.likeCount + 1, liked: !isLiked }));`
- Call `likePost(post.id)` in try/catch.
- On failure: rollback `setIsLiked(prevLiked); setPost(prev => ({ ...prev, likeCount: prevCount, liked: prevLiked }));` and toast error.
- Call `onLikeUpdate?.(post.id, !prevLiked, ...)` on success to sync parent state.

**Comment expand (load comments via getPost):**
- On expand, call `getPost(post.id)` to get PostDetailResponse with embedded comments.
- Set comments from response: `setComments(detail.comments)`.
- Also update post counts from detail response.

**Comment submit:**
- Call `createComment(post.id, { content: commentText })`.
- On success: append returned CommentResponse to comments array, increment commentCount, clear input.

**Comment delete (DEC-019 permission):**
- Add delete button on each comment visible when: `comment.author.id === currentUserId || post.author.id === currentUserId`.
- Call `deleteComment(post.id, comment.id)`.
- On success: filter comment from array, decrement commentCount.

**Post delete:**
- Keep existing delete confirmation UI.
- Replace `postService.deletePost(post.id)` with `deletePost(post.id)` from api/community.
- On success: call `onDelete?.(post.id)`.

**Profile link:**
- `const profileLink = post.author.id === currentUserId ? '/profile' : \`/profile/\${post.author.id}\``.

**Relative time helper:**
- Add a simple inline function or top-level helper: parse `createdAt` ISO string, compute diff from now, return Korean string (방금 전, N분 전, N시간 전, N일 전).

**Import migration:**
- Remove: `import { FeedPostType } from '@/types'`, `import { postService } from '@/services/api/postService'`, `import { memberService } from '@/services/api/memberService'`.
- Add: `import { getPosts } from '@/api/community'`, `import type { PostResponse } from '@/api/community'`.

- Import `useUserStore` from `@/store/useUserStore`.
- Keep existing story imports (already wired from Phase 7).

**State migration:**
- `const [posts, setPosts] = useState<PostResponse[]>([])` -- use PostResponse.
- Add pagination state: `const [page, setPage] = useState(0)`, `const [hasNext, setHasNext] = useState(false)`, `const [isLoadingMore, setIsLoadingMore] = useState(false)`.
- Add error state: `const [hasError, setHasError] = useState(false)`.
- Remove `currentLocation` dependency from useConfigStore (backend getPosts has no location param).

**fetchPosts rewrite:**
- Accept `pageNum: number` parameter.
- If pageNum === 0: `setIsLoading(true)`.
- Else: `setIsLoadingMore(true)`.
- Call `const res = await getPosts({ page: pageNum, size: 10 })`.
- On success: `setPosts(prev => pageNum === 0 ? res.content : [...prev, ...res.content])`, `setHasNext(res.hasNext)`, `setPage(pageNum)`.
- On failure: `setHasError(true)`, toast error.
- Finally: clear loading states.

**Infinite scroll:**
- Add a sentinel div at the bottom of the post list.
- Use `IntersectionObserver` in a useEffect: observe the sentinel, when intersecting and `hasNext && !isLoadingMore`, call `fetchPosts(page + 1)`.
- Use `useRef` for the sentinel element and `useCallback` for fetchPosts.
- Show a small spinner when `isLoadingMore` is true.

**5-state UI for feed list:**
- Loading: existing spinner (Loader2).
- Error: show retry button with '피드를 불러오는데 실패했습니다.' message and '다시 시도' button calling `fetchPosts(0)`.
- Empty: existing empty state message.
- Success: post list with infinite scroll.
- Loading more: small spinner below the list.

**currentUserId for FeedItem:**
- Get from useUserStore: `const profile = useUserStore(s => s.profile)`.
- Pass `currentUserId={profile ? Number(profile.id) : undefined}` to each FeedItem.
- Remove `fetchUserProfile` function and `userProfile` state entirely.

**onDelete handler:**
- `(postId: number) => setPosts(prev => prev.filter(p => p.id !== postId))`.

**onLikeUpdate handler:**
- `(postId: number, liked: boolean, likeCount: number) => setPosts(prev => prev.map(p => p.id === postId ? { ...p, liked, likeCount } : p))`.

**CreatePostModal userProfile prop:**
- Pass `{ nickname: profile?.nickname, profileImageUrl: profile?.profileImageUrl }` instead of full userProfile state.
- Remove `fetchUserProfile` and `userProfile` state.

**onSuccess for CreatePostModal:**
- Call `fetchPosts(0)` to reload from page 0.

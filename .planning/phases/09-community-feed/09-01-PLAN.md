**usePostForm.ts** -- Full rewrite:
- Remove `postService` import. Import `createPost` from `@/api/community` and `uploadImageFlow` from `@/api/upload`.
- Replace `postForm` object state with separate states: `content` (string), `imageFiles` (File[]), `previewUrls` (string[]).
- `handleAddImage(file: File)`: append to `imageFiles`, create object URL via `URL.createObjectURL(file)` and append to `previewUrls`.
- `handleRemoveImage(index: number)`: remove from both arrays by index, revoke the object URL.
- `handleSubmit`: validate imageFiles.length > 0 and content.trim() non-empty (toast warnings in Korean). Upload all images via `Promise.all(imageFiles.map(f => uploadImageFlow(f, 'COMMUNITY_POST')))`, then call `createPost({ content, imageUrls })`. On success: toast '게시글이 등록되었습니다!', reset all state, call `onSuccess?.()`. On failure: toast '등록 중 오류가 발생했습니다.'.
- Return: `{ content, setContent, imageFiles, previewUrls, handleAddImage, handleRemoveImage, isSubmitting, handleSubmit }`.

**PostFormFields.tsx** -- Rewrite props and internals:
- Remove imports of `UserType`, `PostFormData` from `@/types`.
- New props interface: `{ content: string; setContent: (v: string) => void; previewUrls: string[]; onAddImage: (file: File) => void; onRemoveImage: (index: number) => void; authorName?: string; authorAvatar?: string; fileInputRef: React.RefObject<HTMLInputElement | null>; }`.
- Remove `location` input field entirely (backend has no location field).
- Rename caption textarea to content: label stays "Caption" for UI consistency, but bind to `content`/`setContent` props.
- Replace base64 FileReader `onChange` with: `const file = e.target.files?.[0]; if (file) onAddImage(file);`.
- Show preview from `previewUrls[0]` (or first image). Support multiple image display if previewUrls.length > 1 with small thumbnails.
- Use `authorName` and `authorAvatar` props instead of `userProfile.nickname`/`userProfile.avatar`.

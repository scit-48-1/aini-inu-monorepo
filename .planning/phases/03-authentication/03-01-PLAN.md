1. **Create `src/providers/AuthProvider.tsx`** — React Context wrapper around useAuthStore:
   - Interface: `{ isAuthenticated: boolean; isLoading: boolean; login: (email: string, password: string) => Promise<void>; logout: () => Promise<void>; }`
   - Use `createContext` + `use()` (React 19, not `useContext`). Throw if used outside provider.
   - On mount: if refreshToken exists but no accessToken, attempt `refreshToken()` from `api/auth.ts`, store tokens via `useAuthStore.setTokens()`, call `getMe()` from `api/members.ts` to populate profile. Set `isLoading=false` after bootstrap completes.
   - `login()`: call `authApi.login({email, password})`, `setTokens()`, call `getMe()` to populate `useUserStore`, router.push('/dashboard').
   - `logout()`: call `authApi.logout({refreshToken})` (catch and ignore errors), `clearTokens()`, clear profile via `useUserStore.getState().clearProfile()`, router.push('/login').
   - Export `AuthProvider` and `useAuth()` hook.
   - Follow the pattern from RESEARCH.md Pattern 1, but use `use(AuthContext)` instead of `useContext(AuthContext)` per React 19 and the vercel-composition-patterns skill.

2. **Fix `src/api/members.ts` signup return type** — The `signup()` function currently returns `Promise<MemberResponse>`. Per OpenAPI spec, `POST /members/signup` returns `LoginResponse` (with tokens). Change:
   - Import `LoginResponse` from `./auth`
   - Change `signup()` return type: `Promise<LoginResponse>`
   - Change generic: `apiClient.post<LoginResponse>('/members/signup', data)`

3. **Fix `src/store/useUserStore.ts`**:
   - Replace `import { memberService } from '@/services/api/memberService'` with `import { getMe, updateMe } from '@/api/members'` (use new Phase 2 API layer).
   - Add `fetchProfile(force?: boolean)` parameter: when `force=true`, bypass the `hasFetched` guard. This is needed so login/signup can re-fetch profile even if a prior failed attempt set `hasFetched=true`.
   - Replace `logout()`: remove the old `DB_KEY` / `aini_inu_v6_db` localStorage code. Replace with `clearProfile()` that sets `{ profile: null, isAuthenticated: false, hasFetched: false }`.
   - Add `clearProfile()` action (separate from logout, called by AuthProvider).
   - Update `updateProfile` to use `updateMe` from `@/api/members` instead of `memberService.updateMe`.
  </action>

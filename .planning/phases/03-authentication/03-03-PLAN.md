<task type="auto">
  <name>Task 1: Fix session lost on refresh — add skipHydration and explicit rehydrate</name>
  <files>aini-inu-frontend/src/store/useAuthStore.ts, aini-inu-frontend/src/providers/AuthProvider.tsx</files>
  <action>
  **Root cause:** `useAuthStore` persist middleware auto-hydrates at module init time. In Next.js SSR context, `window.localStorage` is unavailable, so `createJSONStorage` returns undefined. Zustand's `persistImpl` detects `!storage` and skips `hydrate()` entirely. When AuthProvider bootstrap runs, `getRefreshToken()` returns null because the store was never hydrated.

  **Fix useAuthStore.ts:**
  Add `skipHydration: true` to the persist config object (alongside the existing `name` and `partialize` fields):
  ```typescript
  {
    name: 'aini-inu-auth',
    partialize: (state) => ({ refreshToken: state.refreshToken }),
    skipHydration: true,
  }
  ```
  This prevents auto-hydration at module init and defers it to an explicit call.

  **Fix AuthProvider.tsx:**
  At the very start of the `bootstrap()` async function (before reading any tokens), call:
  ```typescript
  await useAuthStore.persist.rehydrate();
  ```
  This ensures localStorage data is loaded into the store before `getRefreshToken()` is called. The rehydrate call must happen inside the useEffect (client-side only), where `window.localStorage` is available.

  Important: `rehydrate()` is called on the store itself (`useAuthStore.persist.rehydrate()`), not on the destructured actions. Make sure to import/reference the store object directly for this call.
  </action>

<task type="auto">
  <name>Task 2: Fix login error display — add skipAuth to login API call</name>
  <files>aini-inu-frontend/src/api/auth.ts</files>
  <action>
  **Root cause:** When the login endpoint returns 401 (wrong credentials), `apiClient`'s 401 interceptor in `client.ts` fires because `skipAuth` defaults to `false`. The interceptor calls `handle401` which tries token refresh, fails, then does `window.location.href = '/login'` — a full page reload that destroys React state including the LoginForm's `fieldError` setter. The user sees nothing.

  **Fix auth.ts:**
  In the `login()` function, pass `{ skipAuth: true }` as the third argument to `apiClient.post`:

  ```typescript
  export async function login(data: LoginRequest): Promise<LoginResponse> {
    return apiClient.post<LoginResponse>('/auth/login', data, { skipAuth: true });
  }
  ```

  This tells `client.ts` to NOT intercept the 401 response from the login endpoint. Instead, the 401 will be parsed as a normal error response (ApiError with errorCode), which bubbles up to LoginForm's catch block where it sets `fieldError` state for inline display.

  Do NOT modify `client.ts` — the `skipAuth` option already exists and works correctly. Only `auth.ts` needs the change.

  Also add `{ skipAuth: true }` to the `logout()` function call, since logout should not trigger the 401 refresh cycle either (the token being revoked is the refresh token itself):

  ```typescript
  export async function logout(data: TokenRevokeRequest): Promise<void> {
    return apiClient.post<void>('/auth/logout', data, { skipAuth: true });
  }
  ```
  </action>

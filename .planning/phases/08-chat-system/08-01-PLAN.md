1. Install @stomp/stompjs:
   ```bash
   cd aini-inu-frontend && npm install @stomp/stompjs
   ```

2. Create `src/store/useChatStore.ts` — Zustand store managing:
   - `connectionMode: 'ws' | 'polling' | 'disconnected'` — current connection state
   - `messages: ChatMessageResponse[]` — current room messages
   - `pendingMessages: PendingMessage[]` — optimistic sends awaiting confirmation
   - `rooms: ChatRoomSummaryResponse[]` — room list cache
   - Actions: `setConnectionMode`, `addMessage`, `updateMessageStatus`, `addPendingMessage`, `removePendingMessage`, `markPendingFailed`, `setRooms`, `prependMessages`, `setMessages`, `deduplicateMessage`
   - `PendingMessage` type: `{ clientMessageId: string; content: string; status: 'pending' | 'failed'; sentAt: string }`
   - Message deduplication by `id` or `clientMessageId` using a Set for O(1) lookup (Pitfall 5 from RESEARCH.md)
   - No persistence needed (chat state is ephemeral)

3. Create `src/hooks/useChatWebSocket.ts` — Hook that:
   - Accepts `roomId: number` and `enabled: boolean`
   - On mount (when enabled): creates `@stomp/stompjs` Client with:
     - `brokerURL: \`${process.env.NEXT_PUBLIC_WS_URL || 'ws://localhost:8080'}/ws/chat-rooms/${roomId}\``
     - `connectHeaders: { Authorization: 'Bearer ${token}' }` from useAuthStore
     - `reconnectDelay: 5000`, `heartbeatIncoming: 10000`, `heartbeatOutgoing: 10000`
     - `beforeConnect` callback that refreshes token from useAuthStore (Pitfall 2 from RESEARCH.md)
   - On STOMP connect: subscribes to `/topic/chat-rooms/${roomId}/events`
   - Parses incoming events by type:
     - `CHAT_MESSAGE_CREATED`: calls store.addMessage(event.data.message) + store.deduplicateMessage
     - `CHAT_MESSAGE_DELIVERED`: calls store.updateMessageStatus with deliveredAt
     - `CHAT_MESSAGE_READ`: calls store.updateMessageStatus with readAt
   - Sets connectionMode to 'ws' on connect, 'polling' on disconnect/error
   - On WS failure: falls back to polling via setInterval(5000) that calls `getMessages()` from api/chat.ts and merges into store (DEC-016)
   - Polling and WS are mutually exclusive (anti-pattern from RESEARCH.md)
   - Stores STOMP Client in useRef (not state — anti-pattern from RESEARCH.md)
   - Cleanup: deactivate client + clear polling interval on unmount
   - Returns `{ connectionMode, isConnected: connectionMode === 'ws' }`

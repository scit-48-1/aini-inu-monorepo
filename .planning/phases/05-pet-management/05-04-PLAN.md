**Fix 2 — birthDate empty in edit mode (UAT Test 7):**
The backend Pet entity stores `age` (Integer), NOT `birthDate` (LocalDate). The raw birthDate is lost after conversion. PetResponse therefore cannot return birthDate. The fix is frontend-only:

1. Line 109 — Make birthDate validation conditional on create mode only:
   Change `if (!birthDate)` to `if (!isEdit && !birthDate)` so that edit mode does not block submission when birthDate is empty.

2. Lines 122-128 — In the payload construction, only include birthDate when it has a value. Currently birthDate is always included in the payload object. Change the approach:
   - Build the base payload WITHOUT birthDate
   - Add `birthDate` to payload only if `birthDate` is truthy (user explicitly entered one)
   - This way, edit submissions without a birthDate send null for that field, which the backend PetUpdateRequest treats as "don't change age"

Do NOT modify the backend. The Pet entity intentionally stores computed age, not raw birthDate.

  <name>Task 2: Create API client with envelope unwrap, auth interceptor, refresh queue, and error toast</name>

  <action>
Create `src/api/client.ts` -- the single HTTP gateway for all domain modules. This REPLACES (but does NOT modify or delete) the old `src/services/api/apiClient.ts`.

**Core request function:**
- Base URL: `/api/v1` (Next.js rewrites proxy handles forwarding, configured in Phase 1)
- 8-second timeout via AbortController (same as existing client)
- Auto-attach `Authorization: Bearer {token}` header when `useAuthStore.getState().accessToken` is non-null
- Content-Type: `application/json` for JSON bodies, omit for binary (upload)
- Parse response as JSON, unwrap `ApiResponse<T>` envelope:
  - `success === true` -> return `data as T`
  - `success === false` -> throw `new ApiError(errorCode, message, status)`
- Handle 204 No Content (return `undefined as T`)
- Handle non-JSON responses (throw generic error)

**Auth interceptor (401 handling):**
- On 401 response, check if refresh token exists
- If refreshing already in progress (`isRefreshing` module-level flag), queue the request in a promise array and wait
- If not refreshing, set `isRefreshing = true`, call `POST /api/v1/auth/refresh` with `{ refreshToken }` body
- On refresh success: update tokens via `useAuthStore.getState().setTokens()`, replay all queued requests, clear queue
- On refresh failure (401 or network error): call `useAuthStore.getState().clearTokens()`, show toast "세션이 만료되었습니다", redirect to `/login` via `window.location.href = '/login'`, reject all queued
- The refresh call itself must NOT trigger the 401 interceptor (avoid infinite loop) -- use a `skipAuth` flag

**Error toast (sonner):**
- Import `toast` from `sonner`
- On `ApiError`: show `toast.error(error.message || '요청에 실패했습니다')` with `duration: 3000`
- On network/timeout errors: show `toast.error('네트워크 오류가 발생했습니다', { action: { label: '다시 시도', onClick: retryFn } })` where retryFn re-calls the original request
- On business/validation errors (4xx except 401): toast only, no retry button
- Success toasts are NOT shown by the client (callers handle success UI)
- Export a `suppressToast` option in request config so callers can opt out (e.g., silent background refreshes)

**Exported API:**
```typescript
export const apiClient = {
  get: <T>(endpoint: string, options?) => Promise<T>,
  post: <T>(endpoint: string, body?: unknown, options?) => Promise<T>,
  put: <T>(endpoint: string, body?: unknown, options?) => Promise<T>,
  patch: <T>(endpoint: string, body?: unknown, options?) => Promise<T>,  // NEW
  delete: <T>(endpoint: string, options?) => Promise<T>,
};
```

Options type should extend RequestInit with optional `suppressToast: boolean` and `skipAuth: boolean`.

**Important:** Do NOT use axios -- use native fetch (consistent with existing codebase pattern). Do NOT create a barrel file. Do NOT modify existing `src/services/api/apiClient.ts`.

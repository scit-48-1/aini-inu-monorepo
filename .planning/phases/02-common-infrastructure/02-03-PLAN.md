Functions:
- `getPresignedUrl(data: PresignedImageRequest): Promise<PresignedImageResponse>` -- POST /images/presigned-url
- `uploadToPresignedUrl(token: string, file: File | Blob): Promise<void>` -- PUT /images/presigned-upload/{token}. This is a SPECIAL case: the body is binary (application/octet-stream), NOT JSON. Use fetch directly (not apiClient) since apiClient assumes JSON content-type. Still use the same base URL `/api/v1`. Set Content-Type to the file's MIME type. Parse the response through ApiResponse envelope manually.
- `getImageUrl(key: string): string` -- Returns the URL string `/api/v1/images/local?key=${key}` (no fetch needed, used as img src)
- `uploadImageFlow(file: File, purpose: string): Promise<string>` -- Convenience function that chains: (1) getPresignedUrl with file metadata, (2) extract token from uploadUrl, (3) uploadToPresignedUrl, (4) return imageUrl. This is the function most callers will use. Throws on any step failure.

    2. Props: `isOpen: boolean`, `onClose: () => void`
    3. Search-as-you-type with debounce (300ms, Claude's discretion on interval):
       ```tsx
       const [query, setQuery] = useState('');
       const [results, setResults] = useState<MemberResponse[]>([]);
       const [isSearching, setIsSearching] = useState(false);
       const [hasMore, setHasMore] = useState(false);

       useEffect(() => {
         if (!query.trim()) { setResults([]); return; }
         const timer = setTimeout(async () => {
           setIsSearching(true);
           try {
             const res = await searchMembers(query, { page: 0, size: 20 });
             setResults(res.content);
             setHasMore(res.hasNext);
           } catch { toast.error('검색에 실패했습니다.'); }
           finally { setIsSearching(false); }
         }, 300);
         return () => clearTimeout(timer);
       }, [query]);
       ```
    4. Reset query and results when modal opens (useEffect on isOpen)
    5. Result list items show:

       - Optional: `member.selfIntroduction` truncated to ~50 chars as subtitle
    6. Click on a result: navigate to `/profile/${member.id}` via `useRouter().push()`, close modal
    7. Empty states:
       - No query: show search prompt ("회원 이름으로 검색해보세요")
       - Query but no results: show "검색 결과가 없습니다"
       - Searching: show spinner
    8. Use portal rendering (`createPortal`) consistent with other modals in the project
    9. Style consistent with NeighborsModal: same card shape (rounded-[48px]), same header pattern, same input styling
